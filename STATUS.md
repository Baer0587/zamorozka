> Рабочий документ. Последнее обновление: 2026-06-15

---

## 1. Текущее состояние проекта

Лендинг работает и представляет собой завершённый по структуре одностраничный сайт с каталогом, корзиной и формой заказа, подключённой к n8n-вебхуку. Административная панель (`admin.html`) читает заказы из PostgreSQL через `api.php` и позволяет менять статусы прямо в браузере. Бэкенд-оркестрация (приём заказа → запись в БД → уведомление в Telegram) реализована через n8n-воркфлоу. Сайт в продакшне работает на VPS Beget за Traefik с HTTPS. Параллельно идёт старт миграции на Next.js 15 — пока только в планах, кода нет.

## 2. Что уже работает

**Frontend (index.html)**
- Полный лендинг: Hero с видеофоном, Преимущества, Каталог с табами (4 категории), Процесс приготовления, Форматы заказа (горячее/замороженное), Отзывы, Зона доставки с картой Leaflet, Анонс приложения, Контакты, Footer
- Корзина: добавление товаров, счётчик в шапке, сайдбар с составом, localStorage-персистентность
- Переключение категорий каталога (табы + ARIA)
- Scroll-reveal анимации, адаптив 375→1440px, reduced-motion поддержка
- Кастомная тема Tailwind (cream / terra / bark палитра)
- Шрифт Amatic SC (заголовки) + Cabin (тело)

**Страница заказа (order.html)**
- Показывает корзину из localStorage
- Форма: имя, телефон, адрес, формат, дата/время-слот, комментарий
- Валидация на клиенте
- POST на n8n-вебхук `https://n8n.baern8n.ru/webhook/zamorozka-order`
- Экран успеха / экран ошибки с подробным сообщением

**Админ-панель (admin.html)**
- Авторизация по паролю (сессия в sessionStorage)
- Таблица заказов с полями: №, дата, имя, телефон, адрес, состав, итого, формат, дата/время доставки, статус
- Фильтрация по статусу (Новый / Принят / Готовится / Везём / Доставлен / Отменён)
- Счётчики по статусам в шапке дашборда
- Переключение статусов кнопками действий (PUT на `api.php?action=update_status`)
- Авто-обновление каждые 30 секунд (пауза при скрытой вкладке)
- `noindex, nofollow` в мета-тегах

**AI-чат-виджет**
- Плавающая кнопка чата в правом нижнем углу (`#zam-toggle`, `#zam-window`)
- Запросы на n8n-вебхук `/zamorozka-chat` → OpenAI gpt-4o-mini
- История диалога передаётся в каждом запросе (бот помнит контекст)
- API-ключ OpenAI скрыт на стороне n8n, в браузер не попадает

**Инфраструктура**
- `serve.mjs` — локальный dev-сервер (Node.js, порт 3000, range-запросы для mp4)
- Puppeteer установлен для скриншотов (devDependency)
- `.gitignore` настроен (node_modules, screenshots, package-lock.json)

**Telegram Mini App (Шаг 1 — MVP)**
- Зарегистрирован у BotFather для @zamorozka_spb_bot
- Кнопка «🛒 Магазин» открывает сайт внутри Telegram
- telegram-web-app.js подключён в index.html и order.html
- Определение контекста: Telegram vs браузер (window.IS_TELEGRAM)
- MainButton «Оформить заказ (N шт.)» появляется при наполнении корзины
- Полный цикл заказа работает: Mini App → корзина → форма → n8n → уведомление в бот
- Данные пользователя Telegram передаются в заказ (source, telegram_user)
- Сайт в браузере работает без изменений

**Бэкенд (по описанию, не в этом репо)**
- n8n-воркфлоу принимает заказы → пишет в PostgreSQL → уведомляет Telegram-бот
- `api.php` (на `plemennaya-spb.ru`) — CRUD-слой для админки

## 4. Что в процессе / не доделано

- **Миграция на Next.js 15** — заявлена, но кода нет ни строчки. Текущий проект — это и есть «старая» версия.
- **Реальные фотографии продукции** — все изображения в каталоге — `placehold.co`. Сайт в продакшне должен иметь нормальные фото.
- **Email-форма «Уведомить о запуске приложения»** — форма есть, вебхук для неё не зафиксирован в коде (нет fetch). Скорее всего отправка не работает или реализована где-то отдельно.
- **ВКонтакте ссылка в футере** — `href="#"`, не заполнена.
- **Ссылка MAX в контактах** — ведёт на `max.ru` (главная сайта), а не на конкретный аккаунт бренда.
- **Реальный номер телефона** — на сайте `+7 (812) 000-00-00`, нужно заменить.
- **ТЗ: оладьи, сайт: сырники** — в `tz.txt` блок «Оладьи», в реализации — «Сырники». Расхождение нужно сверить с заказчиком.
- **Нет git-репозитория** — проект не инициализирован в git (только `.gitignore` есть).
- **Mini App Шаг 2** — оплата через Telegram Payments / ЮKassa (не реализовано)
- **Валидация initData** — подпись от Telegram не проверяется на сервере n8n (безопасность)
